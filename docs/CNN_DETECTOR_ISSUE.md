# CNN检测器问题说明

## 问题描述

在BER性能对比中，CNN检测器显示BER≈0.25（接近随机猜测），表明检测完全失效。

## 诊断结果

通过诊断工具分析，发现以下问题：

1. **参数加载正常**：CNN参数文件包含约34K参数，维度和值都在合理范围内
2. **实现不一致**：MATLAB中的CNN实现与Python训练时的实现存在差异
   - 卷积边界处理不同
   - 维度顺序可能不同
   - 归一化方式不一致

## 根本原因

CNN检测器在Python中训练，然后将参数导出到MATLAB进行推理。由于PyTorch和MATLAB的卷积实现细节不同，导致推理结果完全错误。

## 解决方案

**决定：从BER对比中移除CNN检测器，专注于OAMP系列**

理由：
1. 本项目的核心是深度展开（Deep Unfolding）方法，即OAMP-Net
2. CNN是端到端学习方法，与深度展开的研究方向不同
3. 修复MATLAB CNN实现需要大量工作，且不是项目重点
4. OAMP系列（LMMSE → OAMP → OAMP-Net）的对比更有意义

## 替代方案（如需保留CNN）

如果将来需要包含CNN对比，可以：
1. 直接在Python中运行所有检测器，避免MATLAB/Python不一致
2. 重新实现MATLAB版CNN，确保与PyTorch完全一致
3. 使用ONNX等中间格式导出模型

## 更新后的对比方案

| 检测器 | 类型 | 状态 |
|--------|------|------|
| LMMSE | 线性检测 | ✓ 保留 |
| OAMP (修复后) | 迭代检测 | ✓ 保留 |
| OAMP-Net | 深度展开 | ✓ 保留（优化中）|
| CNN | 端到端学习 | ✗ 移除 |
