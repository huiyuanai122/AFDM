# AFDM多符号组帧仿真完整流程说明

## 核心参数
- **SNR点数**: 5个 (0, 5, 10, 15, 20 dB)
- **Monte Carlo次数**: 100次/SNR点
- **每帧符号数**: M = 3个符号
- **总传输次数**: 5 × 100 × 3 = **1500个符号**

## 📊 单个Block的详细流程

### **步骤1: 生成随机信道 (整帧共用)**
```matlab
% 每个block生成一个新的随机信道
l_i = [3, 15, 68, 101]       % 随机时延 (不同block不同)
alpha_i = [1, -1, 0, 0]      % 随机多普勒 (不同block不同)
h_true = [0.3, 0.5, 0.6, 0.5] % 随机增益 (不同block不同)

% 构建DAF域信道矩阵
Heff = Σ h_i · Hi(l_i, alpha_i)  % 1024×1024
```

**关键**: 
- ✅ 每个block信道都不同 (时变特性)
- ✅ 但同一block内的M=3个符号共用这个信道 (准静态假设)

---

### **步骤2: 生成M=3个符号的数据**
```matlab
% 生成3个符号的数据
data_frame = randi([0, 3], 1004, 3)  % 1004×3 (QPSK)

符号1: [data1_1, data1_2, ..., data1_1004]
符号2: [data2_1, data2_2, ..., data2_1004]
符号3: [data3_1, data3_2, ..., data3_1004]
```

**关键**: 
- 3个符号的数据完全独立
- 每个符号占用1004个有效子载波

---

### **步骤3: 三种测试方案并行**

#### **测试1: 单符号基准 (理想情况)**
```
只传输符号1:
  DAF域: x1 (1024×1) 
    ↓ DAF域传输
  y1 = Heff · x1 + w1
    ↓ LMMSE检测
  恢复 data1

特点: 无ISI, 性能最优
```

#### **测试2: 多符号无CPP/CPS (观察ISI)**
```
时域拼接3个符号:
  DAF域: [x1, x2, x3]
    ↓ IDAFT
  时域: [s1, s2, s3]
    ↓ 直接连接 (无保护间隔)
  连续信号: [s1|s2|s3]  ← 符号间有ISI干扰
    ↓ DAF域传输 (添加ISI效应)
  y = H_continuous · [x1;x2;x3] + w
    ↓ 分离3个符号
  [y1, y2, y3]
    ↓ 逐个LMMSE检测
  恢复 [data1, data2, data3]

特点: 有严重ISI, 性能最差
```

#### **测试3: 多符号有CPP/CPS (验证抑制)**
```
时域构建带保护的帧:
  DAF域: [x1, x2, x3]
    ↓ IDAFT
  时域: [s1, s2, s3]
    ↓ 添加CPP/CPS (论文公式5a/5b)
  帧结构: [CPP|s1|CPS][CPP|s2|CPS][CPP|s3|CPS]
            ↑103个  ↑1个  ↑103个  ↑1个  ↑103个  ↑1个
    ↓ 去除CPP/CPS
  纯符号: [s1, s2, s3]
    ↓ DAFT回DAF域
  DAF域: [x1', x2', x3']
    ↓ DAF域传输 (无ISI, 因为CPP/CPS已消除)
  y1 = Heff · x1' + w1
  y2 = Heff · x2' + w2
  y3 = Heff · x3' + w3
    ↓ 逐个LMMSE检测
  恢复 [data1, data2, data3]

特点: ISI被抑制, 性能接近单符号基准
```

## 🔁 100次Monte Carlo的作用

```
Block 1:  信道参数1 → 传输3个符号 → BER1_single, BER1_no, BER1_with
Block 2:  信道参数2 → 传输3个符号 → BER2_single, BER2_no, BER2_with
Block 3:  信道参数3 → 传输3个符号 → BER3_single, BER3_no, BER3_with
...
Block 100: 信道参数100 → 传输3个符号 → BER100_single, BER100_no, BER100_with

统计平均:
  BER_single = mean([BER1, BER2, ..., BER100])
  BER_no_guard = mean([BER1, BER2, ..., BER100])
  BER_with_guard = mean([BER1, BER2, ..., BER100])
```

**目的**: 消除随机性，获得稳定的统计平均性能

## 🎯 关键理解

### **信道时变性**
```
Block 1的信道  ≠  Block 2的信道  ≠  ... ≠  Block 100的信道
   ↓                  ↓                        ↓
 符号1,2,3        符号4,5,6              符号298,299,300
 (共用信道1)      (共用信道2)            (共用信道100)
```

- **时变**: 不同block的信道不同
- **准静态**: 同一block内的3个符号共用同一信道

### **为什么是准静态？**
```
相干时间 T_coherent = 0.833 s
帧持续时间 T_frame = 0.825 s

T_frame < T_coherent  ✓

→ 在一帧(3个符号)的时间内，信道基本不变
```

## 💡 总结

### **完整流程一句话**
> 对5个SNR点，各进行100次Monte Carlo仿真。每次仿真生成一个随机信道，传输3个符号组成的帧，分别测试单符号基准、无CPP/CPS和有CPP/CPS三种方案，统计平均BER。

### **关键数字**
- **总符号数**: 1500 (5 SNR × 100 block × 3 符号)
- **总信道实现**: 500 (5 SNR × 100 block)
- **每个信道传输**: 3个符号 (准静态)
- **测试方案**: 3种 (单符号、无保护、有保护)

### **时变vs准静态**
- **时变**: Block之间信道不同 (100个不同的信道)
- **准静态**: Block内部信道不变 (3个符号共用)
