# 时变信道完整链路闭环改造计划（MATLAB + Python）

## 摘要
本计划用于把项目中的“时变信道”统一为**sequence-level 递推范式**，并打通三条链路：

1. MATLAB oracle 数据导出  
2. MATLAB online 策略评估  
3. Python 训练/评估（RL 与 OAMP/OAMPNet）

目标是解决当前存在的三类不一致：  
- 时变定义分散（不同脚本各写一套）  
- online 评估非真实 sequence rollout（按帧独立采样）  
- state 特征泄漏真值信道（直接使用 `ch.alpha/h/ell`）

---

## 核心改造（决策已锁定）

### 1) 统一时变信道状态机（sequence-level）
新增 MATLAB 公共函数（放在 `matlab/common`）：
- `get_timevary_defaults.m`
- `init_timevary_channel_state.m`
- `step_timevary_channel_state.m`

统一状态定义：
- `state.ell`：默认 sequence 内静态（可选慢变化开关，默认关）
- `state.alpha_t`：AR(1) 递推 + `[-alpha_max, alpha_max]` 裁剪
- `state.h_t`：复高斯 AR(1) 创新 + PDP 加权 + 归一化（防爆）
- `state.pdp`：每条 sequence 固定（默认指数 PDP）

统一超参数（全部有默认值）：
- `rho_alpha`（默认 0.98）
- `rho_h`（默认 0.98）
- `alpha_max`
- `ell_mode`（`static` 默认）
- `pdp_mode`（`exp_fixed_per_sequence` 默认）
- `num_frames`

### 2) MATLAB online 改为真实 sequence rollout
重构 `run_online_policy_oamp_oampnet.m` 主循环为：
- `for sid = 1:num_seq`
- 初始化 sequence 时变状态
- `for tt = 1:num_frames`
- 逐帧构造 `ch_t`、发送 `x_t`、扫描 oracle（仅评估）、RL 单动作执行
- 调用统一 state 特征提取
- 累计 fixed / RL / oracle 指标
- 递推到下一帧

新增输出维度：
- 每帧级：action/reward/ber/regret/switch flag
- 每序列级：switch count/rate、平均 regret、平均 BER
- 总体级：与原 fixed/RL/oracle 可比结果

并新增开关：
- `use_oracle_state = false`（默认）
- 仅 debug 允许真值特征

### 3) State 特征去泄漏并统一 schema（MATLAB/Python 同步）
统一为“在线可观测特征 + 时间上下文”，默认不使用真值 `ch`：
- `Heff_base` 统计（frob、diag_ratio、offdiag_ratio、列能量离散度、cond proxy、band/sparsity proxy）
- 检测可观测 proxy（残差能量、归一化残差、软判决置信度摘要）
- 时间上下文（上一帧动作、上一帧 reward/proxy、最近 K 帧切换计数、归一化 frame index）

实施方式：
- MATLAB 新增 `extract_online_state_features.m`
- Python 更新 `python/rl_c1/features.py`，保证字段顺序和维度一致
- 训练与导出流程统一读取新 `state_dim`

### 4) OAMP/OAMPNet 数据链支持 sequence-aware
`generate_dataset_timescaling_n256.m` 增加模式：
- `mode='iid'`（兼容旧流程）
- `mode='timevary_sequence'`（新增）

`timevary_sequence` 输出新增字段：
- `sequence_id`
- `time_index`
- `dataset_mode`（字符串或枚举，统一口径）
- `timevary_hparams`（`rho_alpha/rho_h/alpha_max/...`）

`python/utils.py` 的 `AFDMDataset` 增强：
- 读取并暴露 sequence metadata
- 支持 frame 独立采样与 sequence-aware split 的索引入口

`train_oampnet_v4.py` 增强：
- `split_mode` 支持 `sample|sequence`（默认 `sequence` 若 metadata 可用）
- train/val/test 按 sequence 划分，避免 frame 泄漏
- 保持当前 frame-wise detector 训练（明确是“时变序列下的单帧检测训练”）

### 5) 统一评估导出字段（MATLAB/Python）
统一逐帧结果字段（long format）：
- `sequence_id`
- `time_index`
- `chosen_action_rl`
- `chosen_action_oracle`
- `chosen_action_fixed`
- `reward_rl`
- `reward_oracle`
- `reward_fixed`
- `ber_rl`
- `ber_oracle`
- `ber_fixed`
- `regret_rl`
- `switch_flag_rl`
- `switch_flag_oracle`（可选）
- `snr_db`
- `detector_target`
- `dataset_mode`

输出格式：
- `.mat`（结构化 + 统计）
- `.csv`（逐帧 long 表，便于画图）

## 验证计划（最小可复现）

### MATLAB：`test_timevary_online_rollout.m`
小规模参数（1~2 分钟）验证：
- 同一 sequence 的 `time_index` 严格递增
- `alpha_t/h_t` 发生演化（非全相等）
- online policy 每帧只选一个动作
- oracle 仅用于评估
- 输出字段齐全且维度正确

### Python：`test_sequence_dataset_pipeline.py`
验证：
- 读取 metadata 成功（`sequence_id/time_index/dataset_mode`）
- sequence split 无泄漏（train/val sequence 集合不交）
- MATLAB 导出 state 维度与 Python 读取一致
- RL eval 的 switch/regret 计算按 sequence 生效
